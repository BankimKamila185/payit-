CASE 1: Phishing Credential Theft & Account Takeover (ATO)
--------------------------------------------------------------------------------
- Mechanism: Scammer steals NetBanking credentials via lookalike pages. Registers GPay on a new device. Bypasses SMS bind via social engineering. Triggers max drain.
- Psychology: 
  * Criminal: Needs fast execution before the victim realizes and locks the SIM card.
  * Victim: Oblivious to the compromise until bank debit alerts arrive.
- Technical Engine Handling:
  * Check Parameters: `device_binding_age_minutes` (<1440m) + `device_signature_hash_mismatch` (True) + `tx_amount_z_score_30d` (>3.0) + `tx_time_of_day_anomaly` (True).
  * Weighted Calculation: 25 (New Device) + 25 (New Binding) + 35 (Amount Spike) + 15 (Odd Hour) = Risk Score: 100.
  * Action: **Hard Block (🔴)** at the NPCI switch. Trigger local device binding suspension.
- Gaps/No Solution: Cloned device fingerprinting where the emulator perfectly mimics the original device metadata, IP location, and transacts during the user's normal daytime hours for a typical average amount.

================================================================================
This section represents the precise mathematical database variables and engineered
features compiled from Stripe Radar, Feedzai, and I4C Suspect Registry specifications.

--------------------------------------------------------------------------------
A. Transactional & Velocity Signals (12 Features)
--------------------------------------------------------------------------------
1.  `tx_amount`: Double.
    The transaction amount in INR.
2.  `tx_timestamp`: Timestamp.
    The UTC epoch millisecond when the transaction was logged at the switch.
3.  `tx_time_delta_last_tx`: Integer.
    $T_{current} - T_{previous}$ in seconds for the same sender ID.
4.  `tx_amount_delta_last_tx`: Double.

B. REAL GRAPH / MULE PATTERNS (beyond chaining — peer-reviewed)
--------------------------------------------------------------------------------
The 8 canonical AML laundering motifs (IBM AMLworld, NeurIPS 2023) — use these
as named patterns, NOT just "chain":
  1. FAN-OUT      — one account → many (k>=2) receivers (distribution/smurfing start)
  2. FAN-IN       — many senders → one account (mule collection / hub)
  3. GATHER-SCATTER — fan-in then fan-out on the SAME account (classic mule)
  4. SCATTER-GATHER — split to many mules, re-aggregate = SMURFING/STRUCTURING
  5. SIMPLE CYCLE — A→B→C→A, money returns to origin = STRONG launder indicator
                    (but high false-positive; signal not proof)
  6. RANDOM       — unstructured layering
  7. BIPARTITE    — two distinct groups (e.g. victims ↔ mules)
  8. STACK        — layered pass-through tiers

Other verified discriminators:
  • TRIADIC TEMPORAL MOTIF (payer→mule→beneficiary, 3-node/3-edge, time-ordered)
    — core mule primitive; TEMPORAL ORDER essential (static triads = false positives,
    legit hubs share topology).
  • TEMPORAL BURSTINESS — mules transact intensively in SHORT windows; legit users
    spread over time. (Strong feature: edges-per-minute concentration.)
  • Methods: PageRank, community detection, GNNs. Real systems: DELATOR
    (GraphSMOTE + multi-task GNN, IEEE BigData 2022), LAS-GNN (temporal motifs on
    180M+ tx graph, ICAIF 2025).
  ⚠️ Caveat: no fixed motif set is THE complete discriminator (one such claim was
    refuted). Use motifs as features feeding a score, not as standalone proof.
